# 墨墨记忆卡 · 表格导入模板

> 用法见 [review-workflow.md](../review-workflow.md) Step 3。AI 每天把 `log/day-NN.md` 整理成 `cards/day-NN.md`（TSV）。
> 每种卡型：把下面的「模板」整段复制 → 粘到墨墨「表格导入」左侧面板（`{{}}` 会标绿）→
> 下载表格 → 把 AI 生成的 TSV 数据行粘进去 → 上传导入。
> **关键：TSV 的列顺序必须和模板里 `{{}}` 出现的顺序一致。** 每种卡型的列顺序已在下方标注。

---

## ① 纠错卡（主力）

亲手写错的搭配/词块/用词。正面"中文意图 + 场景"逼你产出英语，背面给正确 + 你的错 + 原因。

**模板**（粘到墨墨左侧面板）：

```
[P#H1#{{意图}}]
[T#!939393#📍 {{场景}}]
👉 用英语怎么说？
---
✅ [T#B,!36b59d#{{正确}}]
❌ [T#!939393#我写的：{{错误}}]
💡 {{说明}}
```

**列顺序**：`意图 → 场景 → 正确 → 错误 → 说明`

**TSV 示例**：

```tsv
意图	场景	正确	错误	说明
我们得做个决定	讨论推理服务加不加缓存层	we need to make a decision	need do a decision	need to do；decision 跟 make 搭配
临时拉起一个实例	听 Latent Space 讲 Modal serverless	spin up an instance	start a temporary instance	spin up 是工程口语里"拉起服务/实例"的惯用说法
```

## ② 选择题卡

易混词、二选一。墨墨原生交互，做完才出解析。`ans/` 后填正确选项的字母（选项 1=A、2=B、3=C…）。

**模板**：

```
{{题干}}
[Choice#ans/{{答案}}#
- {{选项1}}
- {{选项2}}
- {{选项3}}
]
---
💡 {{解析}}
📍 {{场景}}
```

**列顺序**：`题干 → 答案 → 选项1 → 选项2 → 选项3 → 解析 → 场景`

**TSV 示例**：

```tsv
题干	答案	选项1	选项2	选项3	解析	场景
"这个方案能____延迟风险"用哪个词？	A	mitigate	solve	fix	risk 搭 reduce/mitigate；solve 搭 problem	我写成 this solves the latency risk
```

> 注：上面 `ans/A` 里的 A 是写死在模板里的，但用 `{{答案}}` 占位后由表格列控制，所以模板写成 `ans/{{答案}}`，数据行的「答案」列填 A/B/C。

## ③ 语法 / 概念卡（Q&A）

影响表意的语法点，或想记住的概念。Q&A 是官方推荐"学习效果最好"的原子化卡型。

**模板**：

```
{{问题}}
---
{{答案}}
💡 {{例句}}
📍 {{场景}}
```

**列顺序**：`问题 → 答案 → 例句 → 场景`

**TSV 示例**：

```tsv
问题	答案	例句	场景
when 引导的从句，主句用将来时时从句用什么时态？	从句用一般现在时（主将从现）	I'll restart the server when the cache is full.	我写成 when the cache will be full
```

---

## 通用约定

- **样式只写在模板里，数据行保持纯文本**——最省心、最不易错。
- 颜色码全小写：墨墨绿 `36b59d`、灰 `939393`。
- 一个 `!` = 文字色，两个 `!!` = 背景色；多参数用英文逗号分隔（如 `[T#B,!36b59d#…]`）。
- TSV 粘进 Excel / Numbers / Google Sheets 会自动按制表符分列，**别手动加引号**。
- 三种卡型分别是三次独立的「表格导入」（模板不同）。某周没有某类点，就不导那类。
